# Введение

## Общая информация о проекте

**Название проекта:**  
**Веб-сервис сбора и анализа статистической информации цен на электроэнергию**

**Источник данных:**  
[АТС — Администратор торговой системы оптового рынка электроэнергии и мощности](https://www.atsenergo.ru/)

---

## Цель проекта

Целью проекта является **создание автоматизированного веб-сервиса**, который:
- ежедневно **загружает данные** о выработке и потреблении электроэнергии из открытых источников сайта АТС;
- сохраняет их в базе данных PostgreSQL;
- **анализирует** статистику и рассчитывает сводные показатели;
- **визуализирует результаты** на веб-странице в удобной форме (графики, таблицы, отчёты);
- обеспечивает возможность формирования уведомлений и отчётов на основе полученных данных.

---

## Задачи проекта

1. Разработка **backend-части** для автоматической загрузки, обработки и хранения данных.  
2. Создание **frontend-интерфейса** для визуализации данных (графики, фильтры, таблицы).  
3. Реализация **системы оповещений**, которая информирует о важных изменениях в статистике.  
4. **Математическая обработка** и анализ собранных данных (вычисление средних, максимумов, план/факт и т.д.).  
5. Организация **базы данных** для долговременного хранения и аналитики.

---

## Структура проекта

Проект разделён на несколько модулей, каждый из которых отвечает за свой уровень логики:

| Раздел | Описание |
|--------|-----------|
| **Backend** | Обрабатывает данные, взаимодействует с базой и формирует API для фронтенда. |
| **Frontend** | Визуализирует данные в виде графиков и таблиц. |
| **Database** | Хранит и структурирует данные, поступающие с сайта АТС. |
| **Notifications** | Реализует систему уведомлений и оповещений на основе событий и изменений данных. |
| **Math** | Выполняет математическую обработку данных (средние значения, прогнозы и т.д.). |

---

## Архитектура проекта

1. **Источник данных:** сайт [atsenergo.ru](https://www.atsenergo.ru/), где ежедневно публикуются Excel-файлы с отчётами.  
2. **Скрипт `fetch_data.py`** — автоматически скачивает и подготавливает данные.  
3. **PostgreSQL** — хранит очищенные и обработанные данные.  
4. **Backend (Python)** — выполняет загрузку, преобразование и обработку данных, передаёт их на фронтенд.  
5. **Frontend (HTML, JS)** — отображает данные пользователю в виде интерактивной визуализации.

## Практическое применение

Разработанный сервис может использоваться:
- аналитиками и специалистами энергокомпаний;
- студентами и преподавателями для учебных проектов;
- исследователями в области электроэнергетики;
- разработчиками, изучающими работу с открытыми источниками данных и визуализацией.

---

## Результат работы

- Автоматическая система обновления данных из источника;
- Централизованная база данных для хранения и анализа информации;
- Веб-интерфейс с визуализацией и аналитикой;
- Возможность масштабирования и интеграции с внешними модулями.

## Авторы и используемые технологии

**Технологии проекта:**  
- Backend: Python, Flask, psycopg2 / SQLAlchemy  
- Frontend: HTML, CSS, JavaScript  
- Database: PostgreSQL  
- Средства администрирования: pgAdmin4  
- Среда разработки: Visual Studio Code, GitHub
